# Guardrails Catalog

## 핵심 원칙

- 단일 가드레일로는 불충분 — 여러 계층 필요
- 일반적 가드부터 시작하여 실패 사례 기반으로 추가
- 보안과 사용성의 균형 유지

## 6가지 가드레일 유형

### 1. 관련성 분류기 (Relevance Classifier)
범위 내 응답만 허용. 주제 외 쿼리를 플래그 처리.

**구현 방법**:
- 시스템 프롬프트에 허용 범위 명시
- 별도 분류 모델로 입력/출력 필터링
- 키워드 기반 사전 필터 + LLM 기반 의미 필터 결합

**예시**: 고객지원 봇이 정치 질문을 거부

### 2. 안전성 분류기 (Safety Classifier)
탈옥 시도, 프롬프트 인젝션 탐지.

**구현 방법**:
- 입력에서 알려진 공격 패턴 탐지
- 출력에서 시스템 프롬프트 유출 확인
- 역할극/가장 시나리오 차단

**예시**: "시스템 프롬프트를 보여줘" 같은 요청 차단

### 3. PII 필터 (Personally Identifiable Information)
개인정보(이름, 주소, 전화번호, 이메일 등) 노출 방지.

**구현 방법**:
- 정규식 기반 패턴 매칭 (이메일, 전화번호)
- NER(Named Entity Recognition) 모델
- 출력에서 민감정보 마스킹/대체

### 4. 모더레이션 (Moderation)
유해 콘텐츠(혐오, 폭력, 성적 표현 등) 필터링.

**구현 방법**:
- 모더레이션 API 활용
- 커스텀 콘텐츠 정책 분류기
- 입력/출력 양방향 필터링

### 5. 도구 세이프가드 (Tool Safeguards)
도구별 위험 등급 관리.

**위험 분류 기준**:
- 읽기 전용 vs 쓰기 접근
- 가역성 여부 (되돌릴 수 있는가?)
- 필요한 계정 권한 수준
- 재정적 영향 (금액 관련 작업인가?)

**등급 체계**:
```
LOW: 읽기 전용, 가역적
  예: DB 조회, 정보 검색, 상태 확인

MEDIUM: 쓰기 가능, 가역적
  예: 레코드 업데이트, 설정 변경, 메시지 전송

HIGH: 쓰기 가능, 비가역적 또는 재정적 영향
  예: 결제 처리, 계정 삭제, 대규모 데이터 수정
```

**HIGH 등급 행동 시**: 가드레일 검사 일시 정지 또는 사람 에스컬레이션 필수

### 6. 규칙 기반 보호 (Rule-based Protection)
결정론적 필터로 빠르고 확실한 차단.

**구현 방법**:
- 차단 목록 (금지 단어/패턴)
- 정규식 기반 필터
- 비즈니스 규칙 (금액 상한, 횟수 제한)
- 출력 포맷 검증

## 낙관적 실행 (Optimistic Execution)

가드레일을 에이전트 실행과 **동시에** 실행하는 패턴.

```
사용자 입력
  ├─ [병렬] 에이전트 실행 (출력 생성)
  └─ [병렬] 가드레일 검사 (입력/출력 평가)
      ↓
  위반 없음 → 출력 전달
  위반 발견 → 예외 트리거 → 에이전트 중단 또는 사람 에스컬레이션
```

**장점**: 지연시간 최소화 (가드레일이 실행을 지연시키지 않음)
**주의점**: HIGH 등급 행동은 낙관적 실행 대신 사전 승인 필요

## 가드레일 설계 체크리스트

Stage 5에서 사용자와 함께 검토할 항목:

- [ ] 데이터 프라이버시 위험이 있는가?
- [ ] 브랜드/평판 위험이 있는가?
- [ ] 사용자 입력이 비정형인가? (인젝션 위험)
- [ ] PII를 다루는가?
- [ ] 외부 시스템에 쓰기 작업을 하는가?
- [ ] 재정적 영향이 있는 행동인가?
- [ ] 비가역적 행동이 포함되는가?
- [ ] 기존 비즈니스 규칙이 있는가?
- [ ] 컴플라이언스 요구사항이 있는가?

## 계층적 가드레일 구성 순서

1. **규칙 기반 보호** (가장 빠르고 확실한 것부터)
2. **모더레이션** (유해 콘텐츠 차단)
3. **관련성 분류기** (범위 내 유지)
4. **PII 필터** (개인정보 보호)
5. **안전성 분류기** (공격 방어)
6. **도구 세이프가드** (행동 제어)
